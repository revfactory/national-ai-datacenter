## 어떻게 만들어졌나 — 멀티 에이전트 하네스

본 보고서는 [Claude Code](https://claude.com/claude-code) 위에서 **4명의 리서처 에이전트가 팀으로 자체 조율**하며 작성됐다.

```
                    ┌─── policy   (정부 공식 채널)
                    │
   오케스트레이터 ──┼─── press    (언론 보도 시계열)
   (national-ai-    │
    datacenter-     ├─── industry (인프라·기업·산업 영향)
    research)       │
                    └─── context  (국제 비교·학계·시민사회)

   공통 스킬:        web-research (인용·시간·다층·균형 4대 원칙)
```

### 작업 흐름
1. **Phase 1 — 준비**: 조사 범위 분석, `_workspace/` 생성
2. **Phase 2 — 팀 구성**: 4명 에이전트 동시 스폰, 4개 작업 등록
3. **Phase 3 — 병렬 조사**: 각 에이전트가 작업 claim 후 독립 수행, 교차 영역은 SendMessage로 직접 공유
4. **Phase 4 — 통합 보고서 생성**: 4개 부록을 읽어 통합본 작성
5. **Phase 5 — 다중 라운드 보강**: 4명이 서로 피드백을 주고받으며 5라운드까지 정밀화

### 5라운드 협업 흔적
- 라운드 1: 초기 수집
- 라운드 2: policy가 시계열 공유 → press 시점 보정 / industry 사양 매핑 / context 비교 기준점
- 라운드 3: press 비판 칼럼 5건 공유 → context §4.3.5 신설
- 라운드 4: industry 협약 확정 수치 → context 매트릭스 한국 행 정밀화
- 라운드 5: context AI 기본법 vs EU AI Act 비교, NVIDIA 종속 비판 신설

> **참고**: 본 보고서의 가장 큰 정정 — 초기 작성 시 "첫 공식 발표 = 2024-11-27"로 기재했으나, policy 에이전트의 2차 보강 라운드에서 **2024-09-26 국가AI위 출범식·4대 플래그십 심의·의결**이 진정한 최초 공식화임이 확인되어 종합 보고서·웹사이트 모두 정정됐다.

## 방법론 — 4대 원칙 (web-research 스킬)

모든 사실에 다음 4가지를 강제 적용한다:

1. **인용** — 모든 사실에 출처 URL / 문서명 / 발행일 병기. 출처 없는 진술 금지.
2. **시간** — 발표일 · 기사일 · 자료일 명시. 확정 / 예정 / 검토 단계 명확 구분.
3. **다층** — 1차(정부 공식) → 2차(언론) → 3차(분석·논평) 출처 다층 확보.
4. **균형** — 찬성 · 반대 · 중립 시각 모두 수집. 상충 정보는 양쪽 모두 기록 (삭제 금지).

추가 규칙:
- **익명 인용 금지**, 발언자 직책 포함 필수
- 영문 인용은 **원문 1문장 + 한국어 의역 병기**
- 수치 충돌 시 **시점·출처 모두 기록**
